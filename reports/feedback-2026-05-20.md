### §3 backup 分層矩陣 — sourced(兩官方端點可推)

- Ignition gwbk(docs.inductiveautomation.com/docs/8.1/platform/gateway/
  gateway-backup-and-restore,WebFetch verbatim):涵蓋「all your
  projects, Gateway settings, authentication profiles, Tags, database
  connections, OPC and device connections, alarm pipelines, scripts,
  sequential function charts, reports, and Image Management Library」;
  明文不含「data stored in other programs such as SQL databases, PLC
  programs, other files you manually added to the install directory」。
- Sepasoft(docs.sepasoft.com/articles/user-manual/
  mes-enterprise-challenges-and-solutions,Playwright verbatim):「all
  of the configuration and historical data is stored in one database」;
  多 server 時「configuration and historical data is stored on multiple
  systems and in multiple databases」。

兩端點相乘即可 source 矩陣核心:gwbk 不含 SQL DB 資料 + MES
configuration 存 MES database → gwbk 不含 MES configuration;Ignition
project export(.zip)僅 Ignition project resource。**仍待補**:回饋
矩陣裡「MES Runtime DB backup 無法 import 回另一 gateway」此頁無
verbatim;Enterprise I/E 頁只正面支持「MES File Export 可 import 進任何
裝 MES Platform 3.0 的 gateway」。寫入時 DB-only-backup 那格標待補,
不寫斷言。

### §2 無 script-body API 負面結論 — 回饋寫錯,須改寫為更窄版本

AbstractMESObject Functions(docs.sepasoft.com/articles/user-manual/
abstractmesobject-functions,Playwright verbatim 完整函式表)明列
Artifact Object Functions:`createArtifact`、`getArtifactCount`、
`getAllArtifactNames`、`getArtifactNames`、`getArtifactProperty`、
`getArtifactProperties`、`getArtifactValueByName(artifactName)`、
`getArtifactValueByUUID(artifactUUID)`、`setArtifactValue`。

回饋 §2 原文「AbstractMESObject 完整 method list 沒有 getScript() /
getEventScript() / getArtifact() 之類入口」**與官方頁直接矛盾**:
`getArtifactValueByName` / `getArtifactValueByUUID` / `getArtifactProperty`
明確存在且有文件。回饋此條依 CLAUDE.md self-check(來源是否實際支持
陳述)不通過,**不得照收**。

可 source 的精確負面結論(改寫後):

- AbstractMESObject 文件函式表無 script 專用 accessor(無 `getScript`
  / `getEventScript`);可驗證來源為 abstractmesobject-functions 頁
- Artifact 值經泛用 `getArtifactValueByName` / `getArtifactValueByUUID`
  取得(官方),非 script 專屬 API,且需先知道 artifact name/UUID
- MES Object event script 在 Designer(Project Browser > MES Scripts >
  MES Object Events)編輯(mes-object-events,已 sourced);官方未文件化
  「以 scripting 抽出某 event script 原始碼」的專用 API
- Enterprise Import/Export 是官方 configuration 轉移機制,`Include
  Artifacts` 控制 artifact data 是否隨帶(Enterprise I/E 頁,已 sourced)

§2 進 skill 的形態:窄化的「文件未提供 script-body 專用讀取 API;
script 經 artifact 機制或 Designer/Enterprise I/E」,不寫成回饋的
「完全沒有任何 getArtifact 入口」。
